## FEATURE: AI Lead Generation System for ApollonIA

### Overview
Build a lead generation pipeline that:
1. Takes an Italian city as input
2. Scrapes real estate agencies via Google Places API
3. Enriches leads with emails (from Place Details or website scraping)
4. Sends personalized cold outreach emails via Resend
5. Tracks all activity in SQLite
6. Exposes REST endpoints consumed by a static dashboard

---

### 1. DATABASE SCHEMA

Add these tables to the existing SQLite setup (do not touch existing tables):

```sql
CREATE TABLE IF NOT EXISTS campaigns (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  city TEXT NOT NULL,
  status TEXT DEFAULT 'pending', -- pending | running | paused | completed
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  started_at TIMESTAMP,
  completed_at TIMESTAMP,
  total_found INTEGER DEFAULT 0,
  total_emailed INTEGER DEFAULT 0,
  total_responded INTEGER DEFAULT 0,
  total_booked INTEGER DEFAULT 0
);

CREATE TABLE IF NOT EXISTS leads (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  campaign_id INTEGER REFERENCES campaigns(id),
  agency_name TEXT,
  address TEXT,
  phone TEXT,
  website TEXT,
  email TEXT,
  google_place_id TEXT UNIQUE,
  email_status TEXT DEFAULT 'pending', -- pending | no_email | sent | bounced
  response_status TEXT DEFAULT 'none', -- none | interested | not_interested | booked
  email_sent_at TIMESTAMP,
  responded_at TIMESTAMP,
  notes TEXT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE IF NOT EXISTS agent_logs (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  campaign_id INTEGER REFERENCES campaigns(id),
  lead_id INTEGER REFERENCES leads(id),
  event TEXT NOT NULL, -- e.g. "place_found", "email_found", "email_sent", "response_received"
  detail TEXT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 3. GOOGLE PLACES SCRAPER

Create `services/places_scraper.py`:

- Function `scrape_city(city: str, campaign_id: int, max_results: int = 60)`
- Uses Google Places Text Search API: query = `"agenzia immobiliare {city}"`
- Endpoint: `https://maps.googleapis.com/maps/api/place/textsearch/json`
- Paginate using `next_page_token` (sleep 2s between pages — Google requires this)
- For each place, fetch Place Details to get: name, formatted_address, formatted_phone_number, website, plus check if email exists in the result
- Stay under 3 pages per city (60 results max) to protect free tier
- Skip places already in the leads table (check google_place_id)
- If a website is found but no email, attempt to scrape the website's contact page:
  - Try fetching `{website}/contatti`, `{website}/contact`, `{website}` 
  - Use regex to extract email addresses from HTML
  - Take the first match that is not a generic domain (e.g. skip @gmail, @yahoo, @libero unless nothing else found)
- Write each lead to the DB as it is found
- Log every event to agent_logs
- Update campaigns.total_found as leads are added

### 5. INBOUND RESPONSE WEBHOOK

Create endpoint `POST /leads/response`:

Request body:
```json
{
  "lead_id": 123,
  "response_type": "interested" | "not_interested" | "booked",
  "notes": "optional"
}
```

- Updates lead.response_status and responded_at
- Updates campaigns.total_responded or total_booked
- Logs to agent_logs
- Returns 200 OK

### 6. FASTAPI ROUTES

Create `routers/leads.py` with these endpoints:
POST   /campaigns                    - create campaign (body: {city, max_results})

POST   /campaigns/{id}/start         - trigger scrape + email in background task

GET    /campaigns                    - list all campaigns with stats

GET    /campaigns/{id}               - campaign detail + stats

GET    /campaigns/{id}/leads         - paginated leads for a campaign (query: page, limit, status filter)

GET    /campaigns/{id}/logs          - recent agent_logs for a campaign (limit 100)

PATCH  /campaigns/{id}/status        - pause or resume (body: {status})

POST   /leads/response               - mark a lead response (manual for now)

For `POST /campaigns/{id}/start`: use FastAPI BackgroundTasks to run the scrape and then email sending without blocking the HTTP response.

Add CORS middleware to allow requests from any origin (the static dashboard will call these).
